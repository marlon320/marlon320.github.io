                Meeting 3 2/6/2020
                Fill in the blank apples to apple style questions are good forms for comfortability, can open up to regular type anything. Players could hit a help button to get a starter phrase to use if they show
                PWA/PWD- Progressive web app/dev. Only pulls content as you need it, apple is way behind but apple and android both agreed to support it. Makes a web app look like a native app.
                Phone gap/Cordova. Lots of types to do ios/android compatibility
                The camera is a nice addition, we can have a hardware/software workaround by screenshotting the screen from the camera instead of triggering the camera. Put in design doc as stretch goals. Put in (not a primary goal)
                Concern, if too many people play the game, the game will look too cluttered. Show it later so the client can see and decide how it looks
                For multiplayer connectivity, firebase is the likely option, look at options for connectivity over Bluetooth or the same wifi. Bluetooth is the worst option of these. Microsoft will give nonprofits credits as a “donation” for free API usage.
                Start sketching out how things will look like. Helpful for planning
